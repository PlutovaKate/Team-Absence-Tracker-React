# Project

Small React application for managing team absence requests.

---

# Stack

- React
- Vite
- JavaScript (ES6+)
- Functional Components
- React Hooks
- Plain CSS

---

# Architecture

- Keep components small and reusable.
- One component should have one responsibility.
- Lift state up when multiple components need the same data.
- Pass data through props.
- Keep business logic in parent components when appropriate.

---

# Code Style

- Use functional components only.
- Use useState and other React hooks.
- Write clean and readable code.
- Use meaningful variable and function names.
- Prefer simple solutions over clever ones.
- Keep files organized.

---

# Development Rules

- Explain changes before modifying files.
- Implement features step by step.
- Keep commits small and focused.
- Preserve existing functionality.

---

# What NOT to do

- Do not rewrite working code without a reason.
- Do not change project structure unless necessary.
- Do not install additional libraries.
- Do not implement multiple features at once.
- Do not generate the entire application.

---

# Learning Mode

The developer is learning React.

- Act as a mentor.
- Explain concepts in simple language.
- Give hints before giving the solution.
- Let the developer write the code whenever possible.
- Review code instead of replacing it.
- Explain mistakes and why the fix works.
